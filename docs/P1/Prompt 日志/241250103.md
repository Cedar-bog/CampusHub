| **基本事件流** | 1. **查看详情**：接单者浏览需求详情，确认地点、报酬、时限均符合自身条件。<br>2. **点击接单**：接单者点击页面下方的“我要接单”按钮。<br>3. **确认提示**：系统弹出二次确认框，显示“确认接单后将锁定任务，请确保能够按时完成”。<br>4. **确认承接**：接单者点击“确认”。<br>5. **状态更新**：系统锁定该需求，修改订单状态为**“进行中”**，并记录接单者ID及接单时间。<br>6. **发送通知**：系统向发布者推送应用内通知及短信提醒“您的需求已被XXX接单”。<br>7. **界面跳转**：系统跳转至“我的-进行中的订单”列表，或直接进入与发布者的聊天界面。 |

### 用例规约 3：信用互评

| 项目 | 内容 |
| :--- | :--- |
| **用例名称** | 信用互评 |
| **用例编号** | UC_Rate_03 |
| **参与者** | 需求发布者、服务接单者 |
| **简要描述** | 订单完成后，交易双方在指定时间内对本次合作进行相互评价，系统依据评分重新计算用户信用值。 |
| **前置条件** | 1. 参与者已登录。<br>2. 有一笔**双方共同参与**的订单，且订单状态为**“已完成”**（通常是双方确认完成或系统自动确认完成）。<br>3. 参与者尚未针对此订单提交评价（超时未评则系统默认好评）。 |
| **后置条件** | 1. 双方（或一方超时默认）的评价记录保存至数据库。<br>2. 发布者与接单者的**个人信用评分**根据本次评价结果更新。<br>3. 该订单状态变更为**“已评价”**，彻底归档。 |
| **基本事件流** | 1. **进入评价入口**：订单状态变为“已完成”后，系统在“我的订单”列表对应条目旁显示“去评价”按钮。<br>2. **选择评分**：参与者点击“去评价”，系统展示评价界面。参与者选择星级（1-5星），并在文本框中输入可选评语。<br>3. **提交评价**：参与者点击“提交评价”。<br>4. **保存记录**：系统保存该条评价数据，并标记该参与者“已评”。<br>5. **检测双向完成**：系统后台轮询该订单双方评价状态。<br>6. **结算与更新**：当系统检测到**双方均已提交**评价（或一方超时自动好评生效），系统触发信用算法：<br> - 计算加权平均分。<br> - 更新双方的“信用评分”数值。<br>7. **结果展示**：参与者在个人主页看到信用分微调后的数值及新收到的评价内容。 |
| **替代事件流** | **2a. 超时未评（自动默认）**<br> 2a1. 订单完成后超过7天，用户仍未进行**步骤 2**的操作。<br> 2a2. 系统定时任务自动为该用户生成一条**“系统默认好评”**（5星）记录。<br> 2a3. 直接进入**步骤 5** 的检测逻辑。<br><br>**3a. 提交恶意差评申诉拦截**<br> 3a1. 用户提交评价时打了1星且附带了激烈负面词汇。<br> 3a2. 系统AI内容审核介入，将该评价标记为“存疑”，暂不立刻更新对方信用分。<br> 3a3. 该评价进入**管理员纠纷处理**队列待人工审核。<br> 3a4. 系统提示用户“评价已提交，待审核后生效”。<br><br>**4a. 重复评价拦截**<br> 4a1. 用户试图通过修改URL或后退页面再次提交同一订单的评价。<br> 4a2. 系统校验发现该订单已存在该用户的评价记录。<br> 4a3. 系统拒绝写入，提示“您已完成评价，请勿重复操作”。 |
