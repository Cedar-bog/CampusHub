# AI协作契约

团队名称：实验体#300
日期：2026年4月13日

第一条：AI 使用原则

- AI 是工具，工程师是责任主体
- 所有 AI 输出必须经过人工审查后才能合入项目

第二条：AI 使用范围

- 允许使用 AI 的场景：
  1. 生成代码注释、文档、README
  2. 编写单元测试用例
  3. 生成重复性代码模板
  4. 解释陌生代码片段、排查简单错误
  5. 辅助编写 SQL 查询语句
  6. 重构建议和代码风格优化
- 禁止直接使用 AI 输出（需额外审查）的场景：
  1. 核心业务逻辑
  2. 算法实现
  3. 安全相关代码
  4. 数据库 schema 变更或数据迁移脚本
  5. 网络请求处理
  6. 正则表达式
- 完全禁止使用 AI 的场景：
  1. 生成密码、密钥、Token 或任何凭据
  2. 处理未脱敏的生产数据或敏感个人信息
  3. 逆向工程、破解、绕过安全机制
  4. 生成违反开源协议或第三方条款的代码
  5. 模拟人工提交虚假 commit 或 PR 记录
  6. 代替人工进行 Code Review 的最终判断

第三条：过程记录要求

- 所有与 AI 的交互 Prompt 必须保留日志
- Git commit message 必须标注 [AI-assisted] 或 [Human-written]
- 每阶段填写《AI 协作反思日志》

第四条：代码合入规则

- AI 生成的代码必须通过至少一名团队成员的 Code Review
- 每位成员每个编码阶段至少有 2 个完全手写的核心函数
